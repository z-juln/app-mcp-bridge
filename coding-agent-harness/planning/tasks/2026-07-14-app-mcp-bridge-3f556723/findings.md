# 统一 App MCP Bridge 内部名称 - 发现记录

## 研究发现

### 崩溃提示名称来自内部程序

- 背景：应用展示名已经修改，但系统崩溃弹窗仍显示旧名称。
- 发现：系统使用 App 包中的实际程序名生成崩溃提示，而不是只读取展示名。
- 影响：必须同时修改包目标、App 内程序和系统标识。
- 后续：已完成并通过错误路径验证。

### 历史记录不能伪装成新任务

- 背景：全文替换会误改历史任务编号和审计路径。
- 发现：历史任务身份必须保持不变；产品代码、安装环境和连接配置可以完全迁移。
- 影响：验收搜索明确区分当前产品与只读历史审计记录。
- 后续：无。

## 技术决策

| 决策 | 选择 | 原因 | 替代方案 | 状态 |
| --- | --- | --- | --- | --- |
| 内部程序名 | `app-mcp-bridge` | 与产品和跨平台方向一致 | 只改展示名 | accepted |
| 本地状态迁移 | 安装时保留令牌并迁移目录 | 客户端无需重新配对 | 生成新令牌 | accepted |
| 历史任务身份 | 保留原始编号 | 保证审计链接有效 | 全文强制替换 | accepted |
